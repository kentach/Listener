## サービス名 
### [LISTENER2.0](https://listener-2026.onrender.com/)

## 1. サービス概要
  前職では英会話スクールに勤務しておりました。本アプリは、英会話スクールに通う生徒が「自宅でも音声を活用した英語学習」を行えるようにすることを目的として開発しました。英検5級〜準1級に対応したオリジナル教材を収録しており、生徒は自身のクラスで使用している教材を選択して、音読練習やリスニング学習を行うことができます。教室での学習に加え、自宅でも継続的に英語に触れることで、学習効果の向上を支援します。
---

## 2. このアイデアはどこから生まれたか

### 2-1. きっかけとなった体験
### 【課題① 単語学習以外で英語に触れる機会が圧倒的に不足している】
- このアプリを開発するきっかけとなったのは、生徒との個別面談でした。
  面談を通して、多くの生徒が「単語学習に偏った学習」を行っており、英文を読んだり英語を聞いたりする習慣が十分に身についていないことに気付きました。

### 【課題② リスニング学習のハードルが高い】
- また、生徒との面談では、「リスニングが苦手だが、どのように勉強すればよいかわからない」という相談を受けることがよくありました。
  生徒には授業で使用している教材の音声を活用した学習方法を提案していましたが、音声ファイルはダウンロード形式で提供されており、学習したい音声を探す手間がかかります。
  さらに、同じ音声を繰り返し聞く際には毎回手動で巻き戻す必要があり、学習ツールとして使いづらいという課題がありました。

### 【原因】
- スクールで使用されているオリジナル教材には音声CDやQRコードが付属していましたが、音声ファイルをダウンロードしなければならず、学習したい音声へすぐにアクセスできる環境ではありませんでした。
  また、スマートフォンが学習ツールとして当たり前になった現代において、このような小さな手間は生徒の学習モチベーションを妨げる要因になり得ます。
  特に、自宅学習の習慣が十分に身についていない生徒にとっては、音声学習から離れてしまう原因の一つになっていると考えました。

### 【解決策】
- そこで、生徒がスマートフォンやPCから簡単に教材音声へアクセスできる環境を提供し、自宅でも気軽に音読練習やリスニング学習に取り組めるよう、本アプリを開発しました。
　本アプリでは、英検5級〜準1級に対応した教材を収録しており、生徒は自身のクラスで使用している教材を選択して学習できます。
　また、音声アプリには欠かすことのできない音声の再生・停止・リピートを簡単に行えるため、従来よりも手軽に音声を活用した英語学習を行うことができます。

### 2-2. なぜそれが「気になった」のか
　英語学習者である私の考えでは、英語教材は学習の要です。便利で使いやすいコンテンツがあるからこそ効率的な学習が継続できると感じています。
　教材を通じて得た知識や体験は学習者の血や肉となり、最終的に英語力の土台を形成すると強く思っています。

　前職の英会話スクールで働く中で、学習者が授業外で活用できるオンラインコンテンツがあればいいなとずっと感じていました。
　
　このような状況を踏まえると、どのようなコンテンツを、どのように学習サイクルの中で活用できるかによって、学習者の理解度や定着度が大きく変わると考え、この点に強く関心を持ちました。

### 4-2. 利用シーン
  **①生徒が校舎で自習をするとき**
  **②生徒が自宅で授業の復習でテキストを使うとき**
  **③講師が授業内で音声を使うとき**
  **④小学生の保護者が子供に自宅で英語学習をさせたいとき**

### 4-3. 実際に使われる可能性について(需要・存在確認)
  - 前職の執行役員の方へ本アプリを紹介し、実際の教育現場への導入可能性について相談する予定です。
    また、現場の視点から必要とされる機能や改善点について意見をいただき、今後の機能開発に反映していきたいと考えています。

## 4-4. ユーザーがサービスを導入して利用するまでのイメージ(導入・継続の流れ)
  - まずはテスト運用として、スクールに通う一部の生徒に実際に利用してもらい、利用状況の分析や操作性に関するフィードバックの収集を行います。
  　収集した意見をもとに機能改善やUI/UXの調整を実施し、より学習しやすいアプリへと改善していく予定です。
  　その後、全校の生徒を対象に展開し、自宅学習における音読・リスニング学習の定着を支援することを目指しています。

  - 本アプリの利用を定着させるためには、ユーザーである生徒に「簡単に使えること」と「利用することで学習成果につながること」を感じてもらう必要があります。
    そのため、操作性の向上だけでなく、継続学習を支援する機能の改善や、学習記録の可視化にも取り組んでいきたいと考えています。

### 5-2. それでも自分が作りたい理由
  1. 授業内容の定着を促進できる
    オリジナルテキストと連携したWebアプリを活用することで、生徒は授業で学習した内容を自宅でも復習することができます。
    特に、音読やリスニングは継続的な反復練習が重要ですが、教室での学習時間だけでは十分な練習量を確保できません。本アプリを活用して自宅学習の機会を増やすことで、授業内容の定着と英語力の向上が期待できます。
  2. 学習習慣の形成につながる
    英語力の向上には、継続的な学習習慣が欠かせません。本アプリを活用することで、「宿題として課題を出す → 自宅で音読・リスニング練習を行う → 次回の授業で確認する」という学習サイクルを構築できます。
    これにより、生徒が日常的に英語に触れる機会を増やし、学習習慣の定着を支援することができます。

## 8. このアプリの懸念点とその対策

### 懸念点①

- 何が問題になりそうか：モチベーションが低い生徒にも使ってもらえるか。
- なぜそう思うか：英語学習においては、もともと学習習慣が定着していない生徒も多く、自由利用型の学習ツールでは継続率が低下する可能性があるため。
- そのために考えている対策：
  
- 何が問題になりそうか：全校生徒数千人規模で利用した場合、アプリ上で負荷や不具合が発生する可能性がある。
- なぜそう思うか：音声再生や教材アクセスが同時に集中することで、サーバー負荷の増加やレスポンス遅延が起こる可能性があるため。
　また、実運用規模での負荷テストが未実施の場合、想定外のバグが顕在化する可能性があるため。
- そのために考えている対策：負荷分散を考慮したインフラ設計や、段階的リリース（テスト運用→一部クラス→全校展開）を行う。

### 10-3.キャッチアップ不足の懸念

キャッチアップ不足の段階では以下の懸念点があります。
1. **完成までのハードルが高い**
使用する言語やフレームワークが異なるため、キャッチアップにかかる時間は個人差があるものの、一定の時間を要することが予想されます。
2. **Issueの見積もりが難しい**
「どこまで理解していて、何が分からないのか」「調査にかかる時間」などが不明瞭な場合、開発期間内に完了することが難しくなってしまいます。

例えば、最近注目されているReactやNextについて。
Reactについては**本リリース後に組み込む方向で検討**するのが良いかと考えています。
その後、React単体のキャッチアップを行い、アプリに組み込む技術的な検証をした上で、組み込むかどうかを判断する形が理想かもしれません。
