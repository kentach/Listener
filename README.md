## 1. サービス概要

### 1-1. アプリ開発の背景
- 前職では英会話スクールで英語講師として勤務しておりました。
　本アプリは、前職の英会話スクール専用のアプリで、通う生徒が「音声を活用した英語のトレーニング」を自宅でも行えるようにすることを目的として開発しました。
- 英検5級〜準1級に対応したオリジナル教材を収録しており、生徒はスマホやタブレット一つで、音読練習やリスニング学習を行うことができます。

### 1-2. アプリ開発の主な機能
- 英語の音声の再生・リピート・スキップ・速度変換
- 練習したい音声のお気に入り機能
- 英検のスコア管理機能（Reading/Listening/Speaking/Writing各スコアの推移をグラフで表示）
- 音声に完了・未完了を残せる機能

## 2. このアイデアはどこから生まれたか

### 2-1. きっかけとなった体験
### 【課題① 単語学習以外で英語に触れる機会が圧倒的に不足している】
- このアプリを開発するきっかけとなったのは、生徒との個別面談でした。
  面談を通して、生徒大半が授業外の自宅学習が「単語の暗記のみ」になっており、英文を読んだり英語を聞いたりする習慣が十分に身についていないことに気付きました。

### 【課題② リスニング学習のハードルが高い】
- また、他の生徒との面談で、「リスニングが苦手だが、どのように勉強すればよいかわからない」という相談を受けることがよくありました。
  生徒には授業で使用している教材の音声を活用した「スクリプト音読学習法」を提案していましたが、音声ファイルはダウンロード形式でしか提供されておらず、学習したい音声を探す手間がかかります。また同じ音声を繰り返し聞く際には毎回手動で巻き戻す必要があり、学習ツールとして使いづらいという課題がありました。

### 【原因】
- スクールで使用されているオリジナル教材には音声CDやQRコードが付属していましたが、音声ファイルをダウンロードしなければならず、学習したい音声へすぐにアクセスできる環境ではありませんでした。
- また、スマートフォンが学習ツールとして当たり前になった現代において、このような小さな手間は生徒の学習モチベーションを妨げる要因になり得ます。
- 特に、自宅学習の習慣が十分に身についていない生徒にとっては、音声学習から離れてしまう原因の一つになっていると考えました。

### 【解決策】
- そこで、生徒がスマートフォンやPCから簡単に教材音声へアクセスできる環境を提供し、自宅でも気軽に音読練習やリスニング学習に取り組めるよう、本アプリを開発しました。

### 2-2. なぜそれが「気になった」のか

1. **単語学習だけでは、不十分**
- 単語学習を中心に進めること自体には異論はありません。実際、語彙力は英語力の土台であり、学習初期に重点的に取り組むべき重要な要素です。
  しかし、単語学習だけで英語が思うように上達するわけではありません。
  英語を読めるようになり、聞き取れるようになるためには、単語を覚えることに加えて、実際に英語を読む、音読する、聞くといった学習を継続することが不可欠です。知識として単語を知っているだけでは、実際の英文や音声の中でその単語を瞬時に理解できるようにはなりません。
  特に初学者ほど、単語学習と並行して英語に読む・聞く機会を増やし、「知っている単語」を「使える単語」「理解できる単語」へと変えていくことが重要だと考えています。

2. **生徒に授業を復習するきっかけを作りたい**
- 「2-1. きっかけとなった体験」で述べたように、多くの生徒は授業で学習した長文やリスニング教材を、その場限りで終わらせてしまい、復習する習慣が十分に身についていないのが現状です。
しかし、初学者が英語の土台を築くためには、一度学習した長文やリスニング教材を繰り返し復習することが欠かせません。同じ教材に何度も触れることで、授業で学んだ単語や文法、表現が定着し、英語を読む力や聞く力の向上につながります。
そのため、授業で学習した内容を授業内だけで完結させるのではなく、家庭学習の中でも継続的に復習できる仕組みを整えることが重要であると考えます。

### 4-3. 実際に使われる可能性について(需要・存在確認)
  - 前職の執行役員の方へ本アプリを紹介し、実際の教育現場への導入可能性について相談する予定です。
  - また、現場の視点から必要とされる機能や改善点について意見をいただき、今後の機能開発に反映していきたいと考えています。

## 4-4. ユーザーがサービスを導入して利用するまでのイメージ(導入・継続の流れ)
  1. まずはテスト運用として、スクールに通う一部の生徒に実際に利用してもらい、利用状況の分析や操作性に関するフィードバックの収集を行います。
  2. 収集した意見をもとに機能改善やUI/UXの調整を実施し、より学習しやすいアプリへと改善していく予定です。
  3. その後、全校の生徒を対象に展開し、自宅学習における音読・リスニング学習の定着を支援することを目指しています。

  - 本アプリの利用を定着させるためには、ユーザーである生徒に「簡単に使えること」と「利用することで学習成果につながること」を感じてもらう必要があります。
  - そのため、操作性の向上だけでなく、継続学習を支援する機能の改善や、学習記録の可視化にも取り組んでいきたいと考えています。

## 8. このアプリの懸念点とその対策

### 懸念点①

- 何が問題になりそうか：モチベーションが低い生徒にも使ってもらえるか。
- なぜそう思うか：英語学習においては、もともと学習習慣が定着していない生徒も多く、自由利用型の学習ツールでは継続率が低下する可能性があるため。
- そのために考えている対策：
  
- 何が問題になりそうか：全校生徒数千人規模で利用した場合、アプリ上で負荷や不具合が発生する可能性がある。
- なぜそう思うか：音声再生や教材アクセスが同時に集中することで、サーバー負荷の増加やレスポンス遅延が起こる可能性があるため。
　また、実運用規模での負荷テストが未実施の場合、想定外のバグが顕在化する可能性があるため。
- そのために考えている対策：負荷分散を考慮したインフラ設計や、段階的リリース（テスト運用→一部クラス→全校展開）を行う。

### 10-3.キャッチアップ不足の懸念

キャッチアップ不足の段階では以下の懸念点があります。
1. **完成までのハードルが高い**
使用する言語やフレームワークが異なるため、キャッチアップにかかる時間は個人差があるものの、一定の時間を要することが予想されます。
2. **Issueの見積もりが難しい**
「どこまで理解していて、何が分からないのか」「調査にかかる時間」などが不明瞭な場合、開発期間内に完了することが難しくなってしまいます。

例えば、最近注目されているReactやNextについて。
Reactについては**本リリース後に組み込む方向で検討**するのが良いかと考えています。
その後、React単体のキャッチアップを行い、アプリに組み込む技術的な検証をした上で、組み込むかどうかを判断する形が理想かもしれません。
