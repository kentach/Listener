## 1. サービス概要

### 1-1. アプリ開発の背景
本アプリは、前職の英会話スクール専用のアプリで、
- 英会話スクールに通う生徒が「音声を活用した英語の自宅学習」を気軽に行えるようにしたい
- 授業で終わる学習ではなく、自宅でも継続して復習できる環境を提供したい
という想いを形にしたいと思い、開発しました。英検5級〜準1級対応の自社開発のオリジナル教材を収録しており、生徒はスマホやタブレット一つで、音読トレーニングやリスニングを行うことができます。

### 1-2. アプリ開発の主な機能
- 管理機能（Active Admin）
- 音声の再生・リピート・スキップ・速度変換
- テキストの登録機能（非同期処理）
- 音声のお気に入り登録機能（非同期処理）
- 英検のスコア結果をグラフで表示する機能（Reading/Listening/Speaking/Writing各スコアの推移をグラフで表示）
- 音声に完了・未完了をチェックできる機能（非同期処理）

## 2. このアイデアはどこから生まれたか

### 2-1. きっかけとなった体験
### 課題1. 「生徒の大半が単語学習のみ。英語に触れる機会が圧倒的に不足している。」
このアプリを開発するきっかけとなったのは、生徒との個別面談でした。
面談を通して、生徒大半が授業外での自宅学習が「単語の暗記のみ」になっており、英語を読んだり、聞いたりする習慣が十分に身についていないことに気付きました。

### 課題2. 「リスニング学習のハードルが高い」
生徒との個人面談で、「リスニングが苦手だが、どのように勉強すればよいかわからない」という相談を受けることがよくありました。
私は生徒には授業で使用している教材の音声を活用した「スクリプト音読学習法」を提案していましたが、音声ファイルはCDとダウンロード形式でしか提供されておらず、
学習ツールとして使いづらいという課題がありました。

### 原因
教材には音声CDやQRコードが付属していましたが、音声ファイルをダウンロードしなければならず、学習したい音声へすぐにアクセスできませんでした。
さらに、同じ音声を繰り返し聞くたびに巻き戻しや再生位置の調整が必要であり、継続的な音読・リスニング学習を行うには手間がかかる環境でした。
スマートフォンが学習ツールとして普及した現在、このような小さな手間が学習へのハードルとなり、特に学習習慣が十分に身についていない生徒ほど、英語に触れる機会が減ってしまう原因になっていると考えました。

### 解決策
そこで、生徒がスマートフォンやPCから教材音声へすぐにアクセスできる環境を整え、繰り返し学習しやすい仕組みを提供するため、本アプリを開発しました。
音声のリピート再生や速度調整、スクリプト表示、お気に入り機能などを実装することで、生徒が自宅でも気軽に音読・リスニング学習を継続できることを目指しています。

### 2-2. なぜそれが「気になった」のか

### 1. 単語学習は重要だが、それだけでは十分ではない
単語学習は英語力の土台であり、学習初期には欠かせない要素です。
しかし、単語を覚えるだけでは、実際の英文や音声の中でその意味を瞬時に理解できるようにはなりません。
私自身も英語を学ぶ中で、「知っている単語」と「使える・理解できる単語」には大きな差があることを実感しました。
その差を埋めるためには、単語学習と並行して、英語を読む・音読する・聞くといった学習を継続することが重要だと考えています。

### 2. 授業で学んだ内容を、家庭学習につなげたい
実際の教育現場では、多くの生徒が授業で学習した長文やリスニング教材を、その場限りで終わらせてしまっています。
しかし、英語力を伸ばすためには、一度学んだ教材を繰り返し復習することが欠かせません。
同じ教材に何度も触れることで、授業で学んだ単語や文法、表現が定着し、読む力・聞く力の向上につながります。
そのため私は、授業で終わる学習ではなく、自宅でも継続して復習できる環境を提供したいと考え、本アプリを開発しました。

## 4. 本当に解決したい課題は何か
このアプリで本当に解決したい課題は、**「生徒が授業外で英語に触れる時間が圧倒的に不足していること」**です。

英会話スクールでは週に1回、60〜90分程度しか授業がありません。その限られた時間だけでは、英語力を伸ばすために必要な学習量を確保することは難しいと感じていました。実際に前職で面談を行う中でも、多くの生徒は自宅学習が単語暗記だけになっており、

- 英文を読む
- 英語を音読する
- 英語を聞く

という学習がほとんど行われていませんでした。

また、教材の音声はCDやダウンロード形式で提供されていたため、

- 学習したい音声にすぐアクセスできない
- 繰り返し再生しづらい
- スマートフォンで気軽に学習できない

という理由から、授業内容を復習する機会も失われていました。

そこで本アプリでは、「授業で終わる英語学習」を「家庭でも継続できる英語学習」へ変えることを目的としています。
音声・スクリプト・学習記録を一つのアプリにまとめることで、生徒が毎日少しずつ英語に触れる習慣を作り、授業内容の定着と英語力向上を支援します。

## 9. このアプリの懸念点とその対策

### 懸念点①

- 何が問題になりそうか：モチベーションが低い生徒にも使ってもらえるか。
- なぜそう思うか：英語学習においては、もともと学習習慣が定着していない生徒も多く、自由利用型の学習ツールでは継続率が低下する可能性があるため。
- そのために考えている対策：現場の講師に協力してもらいアプリを促進する

### 懸念点②

- 何が問題になりそうか：全校生徒数千人規模で利用した場合、アプリ上で負荷や不具合が発生する可能性がある。
- なぜそう思うか：音声再生や教材アクセスが同時に集中することで、サーバー負荷の増加やレスポンス遅延が起こる可能性があるため。
　また、実運用規模での負荷テストが未実施の場合、想定外のバグが顕在化する可能性があるため。
- そのために考えている対策：負荷分散を考慮したインフラ設計や、段階的リリース（テスト運用→一部クラス→全校展開）を行う。

## 11. 技術スタック（手段としての技術）

### 11-1. 使用予定の技術

- DB：PostgreSQL
- デプロイ先：Render
- 使用予定ライブラリ：devise（新規登録・ログイン）、gretel（パンくず）、chartkick(グラフ化)、activeAdmin(ユーザー管理)

### 11-2. 技術選定の理由

- なぜこの技術を使うのか：今回Ruby on Railsを選択した理由は、プログラミングスクールRUNTEQでメインで学習したという経緯と、プログラミング初学者でも使いやすいGemが多く備わっていたから。
- 今回チャレンジしたい点：実際の業務で使えるかどうかを確認したかったので、管理画面まで実装した。
- 不安な点：不安な点 実際に全校の生徒が使うとなった場合、アプリ上でどのような不具合が発生するか懸念点。
